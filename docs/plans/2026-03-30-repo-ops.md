# リポジトリ運用整備 実装計画

## 概要

dotfiles リポジトリの自動化・品質管理基盤を整備する:

- **CI/CD**: GitHub Actions で flake 検証 + lint、週次 flake.lock 自動更新
- **タスクランナー**: Justfile で日常操作（switch, update, lint, clean）を一元管理
- **Git Hooks**: cachix/git-hooks.nix で pre-commit hooks を Nix ネイティブに管理
- **.gitignore**: 不要ファイルの追跡除外

**出典**: [Issue #69](https://github.com/yu-sz/life/issues/69)

---

## 決定事項

| 項目 | 決定 | 備考 |
|------|------|------|
| CI Nix インストーラ | **cachix/install-nix-action@v31** | upstream Nix。ADR 準拠。DeterminateSystems 版は 2026-01-01 に upstream 廃止済み |
| CI runner | **ubuntu-latest** | コスト $0。darwin 出力は nix flake check が自動スキップ |
| flake.lock 更新 | **DeterminateSystems/update-flake-lock@v28** | cachix installer と併用可能（[実績](https://github.com/DeterminateSystems/update-flake-lock/pull/62)） |
| flake.lock 健全性 | **DeterminateSystems/flake-checker-action@v9** | Nix インストーラではない。upstream Nix で動作 |
| タスクランナー | **Just** | [NixOS & Flakes Book](https://nixos-and-flakes.thiscute.world/best-practices/simplify-nixos-related-commands) 推奨。devShell に配置 |
| Git Hooks | **cachix/git-hooks.nix** | Nix ネイティブ。flake-parts 不要（[README](https://github.com/cachix/git-hooks.nix#readme)） |
| nixfmt フック名 | `hooks.nixfmt.enable = true` | nixpkgs-unstable の nixfmt = v1.2.0（RFC スタイル） |
| マルチシステム対応 | `forEachSystem` で `checks`, `devShells`, `formatter` を生成 | `["aarch64-darwin" "x86_64-linux"]`。CI の Linux runner で checks が動作するため |
| permissions | `contents: write` + `pull-requests: write` | update-flake-lock の PR 作成に必要 |
| GITHUB_TOKEN 制約 | 作成した PR は CI をトリガーしない | 初期段階では必須チェック未設定のため問題なし |

## 設計: flake.nix 変更点

### 差分サマリ

| 変更 | 説明 |
|------|------|
| inputs に `git-hooks` 追加 | `github:cachix/git-hooks.nix`、nixpkgs follows |
| outputs に `self`, `git-hooks` 追加 | checks と devShell 間の参照に必要 |
| `supportedSystems` + `forEachSystem` 追加 | `["aarch64-darwin" "x86_64-linux"]` |
| `devShells` を `forEachSystem` 化 | `enabledPackages` + `shellHook` を git-hooks から継承 |
| `formatter` 追加 | `nix fmt` を有効化 |
| `checks` 追加 | nixfmt, statix, deadnix, shellcheck のフック |

### 最終形

```nix
{
  description = "suta-ro's dotfiles";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";
    nix-darwin = {
      url = "github:nix-darwin/nix-darwin";
      inputs.nixpkgs.follows = "nixpkgs";
    };
    home-manager = {
      url = "github:nix-community/home-manager";
      inputs.nixpkgs.follows = "nixpkgs";
    };
    nix-homebrew.url = "github:zhaofengli/nix-homebrew";
    git-hooks = {
      url = "github:cachix/git-hooks.nix";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs =
    {
      self,
      nixpkgs,
      nix-darwin,
      home-manager,
      nix-homebrew,
      git-hooks,
      ...
    }:
    let
      sharedOverlays = [
        # TODO: nixpkgs-unstable に direnv 修正 (PR #502769) が到達したら削除
        (_final: prev: {
          direnv = prev.direnv.overrideAttrs (old: {
            postPatch = (old.postPatch or "") + ''
              substituteInPlace GNUmakefile --replace-fail " -linkmode=external" ""
            '';
          });
        })
        (import ./nix/overlays)
      ];

      supportedSystems = [
        "aarch64-darwin"
        "x86_64-linux"
      ];
      forEachSystem = nixpkgs.lib.genAttrs supportedSystems;

      mkDarwinConfig =
        {
          username,
          system ? "aarch64-darwin",
        }:
        nix-darwin.lib.darwinSystem {
          inherit system;
          specialArgs = { inherit username; };
          modules = [
            {
              nixpkgs.overlays = sharedOverlays;
              nixpkgs.config.allowUnfreePredicate =
                pkg:
                builtins.elem (nixpkgs.lib.getName pkg) [
                  "claude-code"
                ];
            }
            ./nix/hosts/darwin-shared.nix
            nix-homebrew.darwinModules.nix-homebrew
            home-manager.darwinModules.home-manager
            {
              home-manager = {
                useGlobalPkgs = true;
                useUserPackages = true;
                users.${username} = import ./nix/home;
                extraSpecialArgs = { inherit username; };
              };
            }
          ];
        };
    in
    {
      darwinConfigurations = {
        "yutanoMacBook-Pro" = mkDarwinConfig { username = "suta-ro"; };
      };

      devShells = forEachSystem (system:
        let
          pkgs = import nixpkgs {
            inherit system;
            overlays = sharedOverlays;
          };
          inherit (self.checks.${system}.pre-commit-check) shellHook enabledPackages;
        in
        {
          default = pkgs.mkShell {
            inherit shellHook;
            buildInputs = enabledPackages ++ (with pkgs; [
              just
            ]);
          };
        }
      );

      formatter = forEachSystem (system: nixpkgs.legacyPackages.${system}.nixfmt);

      checks = forEachSystem (system: {
        pre-commit-check = git-hooks.lib.${system}.run {
          src = ./.;
          hooks = {
            nixfmt.enable = true;
            statix.enable = true;
            deadnix.enable = true;
            shellcheck.enable = true;
          };
        };
      });

      # Linux (standalone home-manager) — 将来用
      # homeConfigurations."<user>@ubuntu" = home-manager.lib.homeManagerConfiguration {
      #   pkgs = import nixpkgs { system = "x86_64-linux"; overlays = sharedOverlays; };
      #   modules = [ ./nix/home ];
      #   extraSpecialArgs = { username = "<user>"; };
      # };
    };
}
```

## 設計: Justfile

```just
default:
    @just --list

# darwin-rebuild switch を実行
switch:
    darwin-rebuild switch --flake .

# flake.lock を更新して switch
update:
    nix flake update
    just switch

# nixfmt + statix でリントチェック
lint:
    nix fmt -- --check .
    statix check .

# Nix store のガベージコレクション
clean:
    nix-collect-garbage -d
```

> `drs`（`config/zsh/lazy/nix.zsh:9`）との関係: `drs` はシェルのどこからでも使える汎用エイリアス、`just switch` はリポジトリルートでの運用向け。共存で問題なし。

## 設計: CI ワークフロー

### ci.yml

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: cachix/install-nix-action@v31
        with:
          extra_nix_config: |
            access-tokens = github.com=${{ github.token }}

      - uses: DeterminateSystems/flake-checker-action@v9

      - name: nix flake check
        run: nix flake check

      - name: nixfmt
        run: nix fmt -- --check .

      - name: statix
        run: nix develop --command statix check .

      - name: shellcheck
        run: nix develop --command shellcheck scripts/install.sh scripts/bootstrap.sh scripts/setup/*.sh
```

### update-flake.yml

```yaml
name: Update flake.lock

on:
  workflow_dispatch:
  schedule:
    - cron: "0 0 * * 1"

jobs:
  update:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4

      - uses: cachix/install-nix-action@v31
        with:
          extra_nix_config: |
            access-tokens = github.com=${{ github.token }}

      - uses: DeterminateSystems/update-flake-lock@v28
        with:
          pr-title: "chore(nix): update flake.lock"
          pr-body: |
            Automated weekly update of flake.lock.
```

## 意思決定の根拠（ADR 記載内容）

### 決定 1: CI の Nix インストーラ — cachix/install-nix-action

**比較対象**: cachix/install-nix-action vs DeterminateSystems/nix-installer-action

| 観点 | cachix/install-nix-action | DeterminateSystems/nix-installer-action |
|------|--------------------------|----------------------------------------|
| インストールする Nix | **upstream Nix** | **Determinate Nix**（独自フォーク） |
| GitHub Stars | 666 | 230 |
| upstream Nix 対応 | ネイティブ | 2026-01-01 に廃止済み |
| テレメトリ | なし | デフォルト有効 |
| 速度 (Ubuntu) | ~5s | ~17s |

**採用理由**:
- [ADR: Nix パッケージ管理](../adr/2026-03-28-nix-package-management.md) で upstream Nix を選択済み → DeterminateSystems 版は選択不可
- DeterminateSystems 版は [2026-01-01 に upstream サポートを廃止](https://determinate.systems/blog/installer-dropping-upstream/)
- NixOS/nixpkgs 公式リポジトリが cachix 版を使用
- [nix.dev 公式 CI ガイド](https://nix.dev/guides/recipes/continuous-integration-github-actions.html) が推奨
- [NixOS Discourse ベンチマーク](https://discourse.nixos.org/t/which-github-nix-installer-action-is-faster/25878): cachix の方が高速

**補足**: DeterminateSystems/flake-checker-action と update-flake-lock は Nix インストーラではなく、upstream Nix で動作する。

### 決定 2: タスクランナー — Just

**比較対象**: Just vs Makefile vs go-task vs nix run

| 観点 | Just | Makefile | go-task | nix run |
|------|------|----------|---------|---------|
| .PHONY 宣言 | 不要 | 全タスクに必要 | 不要 | N/A |
| インデント | スペース可 | **タブ必須** | YAML | N/A |
| タスク一覧 | `just --list` | 標準機能なし | `task --list` | なし |
| Nix エコシステム採用 | **高い** | 中 | **ほぼゼロ** | ニッチ |
| 起動速度 | 即時 | 即時 | 即時 | flake 評価あり |

**採用理由**:
- [NixOS & Flakes Book](https://nixos-and-flakes.thiscute.world/best-practices/simplify-nixos-related-commands) が明確に推奨
- 著者 ryan4yin が [nix-config](https://github.com/ryan4yin/nix-config/blob/main/Justfile) で 50+ レシピ運用
- Make はビルドシステム、Just はコマンドランナー（[公式](https://just.systems/man/en/)）。dotfiles の用途に Just が合致
- go-task: Nix コミュニティでの採用ゼロ（GitHub 検索 `Taskfile.yml + nix + flake` = 0件）
- nix run: 1タスク 5-10行 vs Just 2-3行で冗長。flake 評価コストもかかる

### 決定 3: Git Hooks — cachix/git-hooks.nix

**比較対象**: cachix/git-hooks.nix vs pre-commit framework vs lefthook

| 観点 | cachix/git-hooks.nix | pre-commit.com | lefthook |
|------|---------------------|----------------|----------|
| Stars | 792 | 15,093 | 7,869 |
| ランタイム依存 | Nix のみ | **Python 必須** | Go バイナリ |
| `nix flake check` 統合 | **YES** | NO | NO |
| `nix develop` 自動セットアップ | **YES** | 手動 | 手動 |
| nixfmt/statix ビルトイン | **YES** | NO | NO |
| luacheck/stylua ビルトイン | **YES** | YES（独自 DL） | NO |
| バージョン管理 | flake.lock 固定 | hook リポジトリ依存 | 自前 |
| 設定の二重管理 | なし | `.pre-commit-config.yaml` + Nix | `lefthook.yml` + Nix |

**採用理由**:
- Hooks が Nix derivation → バージョンが `flake.lock` で完全固定。Nix の再現性哲学と整合
- `nix develop` で自動セットアップ → オンボーディングコストゼロ
- `nix flake check` で CI と hooks が統一検証
- pre-commit は Python 依存。`language: system` は公式で "unsupported" と命名（[v4.4.0+](https://pre-commit.com/)）
- lefthook は Nix 統合なし、設定の二重管理
- 将来の Lua hooks（luacheck, stylua）も `hooks.luacheck.enable = true;` の1行（[ビルトイン hooks 一覧](https://github.com/cachix/git-hooks.nix#built-in-hooks)）
- 旧名 `pre-commit-hooks.nix` → 2024年頃 `git-hooks.nix` にリネーム

## 実現可能性レビュー

| 懸念 | 検証結果 | 根拠 |
|------|---------|------|
| Linux runner で nix flake check が動くか | darwin 出力は自動スキップ。`checks` を `x86_64-linux` にも生成すれば OK | [Nix マニュアル](https://nix.dev/manual/nix/latest/command-ref/new-cli/nix3-flake-check) |
| git-hooks.nix が flake-parts なしで動くか | 動く。README に standalone 例あり | [git-hooks.nix README](https://github.com/cachix/git-hooks.nix#readme) |
| enabledPackages パターンは正しいか | `inherit (self.checks.${system}.pre-commit-check) shellHook enabledPackages;` が公式例 | 同上 |
| update-flake-lock が cachix installer で動くか | 動く。内部で `nix flake update` 相当を実行するだけ。併用実績あり | [PR #62](https://github.com/DeterminateSystems/update-flake-lock/pull/62) |
| GITHUB_TOKEN で PR 作成できるか | できる。`permissions` 明示が必要 | [peter-evans/create-pull-request](https://github.com/peter-evans/create-pull-request#token) |
| nixfmt フック名は何か | `hooks.nixfmt.enable = true`（nixpkgs-unstable の nixfmt = v1.2.0 RFC スタイル） | nixpkgs ソース確認済み |
