# Plans フォーマットリファレンス

## 構造テンプレート

```markdown
# [タイトル] 実装計画

## 概要

[施策の箇条書き]

**出典**:

- [ADR: タイトル](../adr/YYYY-MM-DD-slug.md)

---

## 決定事項

| 項目 | 決定               | 備考 |
| ---- | ------------------ | ---- |
| ...  | **太字で決定内容** | 補足 |

---

## 設計: [コンポーネント名]

\`\`\`nix

# コピペ可能な完成形コード

\`\`\`

---

## 実装手順

### Phase 1: [フェーズ名]

- [ ] 1-1: [タスク]
- [ ] 1-2: [タスク]

### Phase 2: [フェーズ名]

- [ ] 2-1: [タスク]

---

## 変更対象ファイル一覧

| ファイル       | Phase 1  | Phase 2 |
| -------------- | -------- | ------- |
| `path/to/file` | 変更内容 | -       |
```

## 必須セクション

1. **概要**: ADR 出典リンクを含める（ADR がない施策は出典なしで可）
2. **決定事項**: 3列テーブル（項目・決定・備考）。実装上の選択を一覧化
3. **設計**: `## 設計: [名前]` で複数配置可。コードはコピペ可能な完成形
4. **実装手順**: Phase 分割 + チェックリスト `- [ ] N-M: タスク`
5. **変更対象ファイル一覧**: Phase 横断でどのファイルがどう変わるか

## 決定事項テーブル

常に3列。決定内容は太字:

```markdown
| 項目   | 決定                     | 備考                                       |
| ------ | ------------------------ | ------------------------------------------ |
| Stylix | **完全削除**             | autoEnable = false + targets 空 → 影響ゼロ |
| nh     | **home.packages に追加** | devShell ではなくユーザーパッケージ        |
```

## 設計セクション

言語タグ付きコードブロック: `nix`, `zsh`, `just`, `yaml`, `bash` 等。
ファイルパスをコメントで明示:

````markdown
## 設計: nix.settings

```nix
# nix/hosts/darwin-shared.nix
nix.settings = {
  experimental-features = [ "nix-command" "flakes" ];
  warn-dirty = false;
};
```
````

## チェックリスト

Phase 番号 + 連番で一意に識別:

```markdown
### Phase 1: Stylix 削除 + flake-parts 移行

- [ ] 1-1: `flake.nix` から `stylix` input を削除
- [ ] 1-2: `nix/hosts/darwin-shared.nix` から stylix ブロックを削除
- [ ] 1-3: `nix flake check` で検証
```

## 予実差異（実装中に追記）

想定と実際に差異が発生した場合、チェックリスト直後にブロック引用で追記:

```markdown
- [x] 1-3: `nix flake check` で検証

> **予実差異**: 新規ファイルは `git add` が必要だった（Nix flake は Git 追跡ファイルのみ参照）。`git add` 後に再実行で解決。
```

## 実現可能性レビュー（オプション）

技術的な懸念を事前検証した場合に追加:

```markdown
## 実現可能性レビュー

| 懸念                                  | 検証結果 | 根拠                               |
| ------------------------------------- | -------- | ---------------------------------- |
| flake-parts で git-hooks.nix は動くか | 動く     | スタンドアロン統合が可能（README） |
```
